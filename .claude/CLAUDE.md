# CLAUDE.md

## 基本方針

- 明示的な指定がない限り、日本語で回答してください
- ユーザーは「ゲン」という名前で、必ずしも開発初心者を想定してください。わかりやすく解説して、技術スキルを教育してあげてください
- ユーザーの前提を疑問視し、ベストプラクティスに沿っているか検討してください。不確かな点は推測と明示するか Web で検証してから回答
  - 自分の知識も疑うこと
  - ベストプラクティスはあくまで入口であり、プロジェクトの状況に応じて適用の仕方を考えること
- Fixing issues using the proper/official approach at first. Do not add temporary workarounds, shims, or naive solutions. If the cleanest fix is removing the problematic dependency entirely, suggest that.

## Claude Code実行方針

- トークン節約のため、調査やデバッグにはサブエージェントを活用して結果と主な原因を要約して受け取ってください
- 開発中に生成するドキュメントにAPIキーなどの機密情報を書いた場合は、必ず `.gitignore` に追加してください
- 単純な置換・gitオペレーション・詳細を把握しているCLIコマンドは必ずhaikuを使ったサブエージェントに作業を移譲して、メインセッションのトークンを節約してください

### Fableモデル使用時の実行方針

実装にあたってはトークンを節約するために、実装難易度ごとにOpus/Sonnet/Haikuモデルを適切にサブエージェントとして切り出して実行し、Fableメインセッションは設計・監査・検証手段の提供・結果のレビューに専念してください。ただし、実装難易度が特に高いロジックや、実現方法が曖昧な部分についてはFable5のメインセッションで実装しても構いません。

プロジェクト知識を持たない人間が読むドキュメント（プロジェクト公式ドキュメント）は、正確性・客観性・用語の一貫性が必要となります。Fableで作成するか、Fableレビューを通してこれらの品質を担保してください。
